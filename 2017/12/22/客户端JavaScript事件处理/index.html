querySelector 和 querySelectorAll

1.querySelector 和 querySelectorAll 方法是 W3C Selectors API 规范中定义的。他们的作用是根据 CSS 选择器规范，便捷定位文档中指定元素.

2.querySelector和querySelectorAll的参数须是符合 css selector 的字符串。不同的是querySelector返回的是一个对象，querySelectorAll返回的一个集合(NodeList)。

focus blur 和 focusin focusout

1.focus blur 不支持冒泡， focusin focusout 支持冒泡

2.事件执行顺序为focusin &gt; focus &gt; focusout &gt; blur

mouseover mouseout 和 mouseenter mouseleave

1.mouseover,mouseout 冒泡，mouseenter, mouseleave 不冒泡

2.与 mouseenter 事件不同，不论鼠标指针穿过被选元素或其子元素，都会触发 mouseover 事件。只有在鼠标指针穿过被选元素时，才会触发 mouseenter 事件。

3.与 mouseleave 事件不同，不论鼠标指针离开被选元素还是任何子元素，都会触发 mouseout 事件。只有在鼠标指针离开被选元素时，才会触发 mouseleave 事件。

addEventListener attachEvent on

### Mozilla中：



target： 文档节点、document、window 或 XMLHttpRequest。 
type： 字符串，事件名称，不含“on”，比如“click”、“mouseover”、“keydown”等。 
listener ：实现了 EventListener 接口或者是 JavaScript 中的函数。 
useCapture ：是否使用捕捉，一般用 false 。例如：document.getElementById("testText").addEventListener("keydown", function (event) { alert(event.keyCode); }, false); 


### IE中：



target： 文档节点、document、window 或 XMLHttpRequest。 
type： 字符串，事件名称，含“on”，比如“onclick”、“onmouseover”、“onkeydown”等。 
listener ：实现了 EventListener 接口或者是 JavaScript 中的函数。 例如：document.getElementById("txt").attachEvent("onclick",function(event){alert(event.keyCode);}); 
attachEvent不需要指定capture/bubble的参数, 因为在windows IE环境下都是使用Bubble的模式



### 不同点：

    1.IE8以后都支持标准事件模型addEventListener，IE9之前不支持，支持attachEvent
       2.addEventListener为3个参数，支持时间冒泡和捕获，attachEvent为两个参数
        3.addEventListener第一个参数为事件类型不带’on’，而attachEvent需要带’on’
    4.addEventListener中事件处理函数中的this指向事件目标，而attachEvent事件处理函数中的this指向window。
    5.调用顺序：对象属性和HTML属性注册一直优先调用；addEventListener按注册顺序调用；attachEvent顺序随机调用

### on:

1. On绑定多个相同事件会被覆盖，addEventListener可以多次绑定同一个事件并且不会覆盖上一个事件。
2.addEventListener可以使用事件捕获阶段触发事件发生，而on做不到

### 事件传播

事件捕获 -> 执行目标处理程序 -> 事件冒泡 -> 执行父级目标处理程序 -> document -> window

事件捕获只能用于以addEventListener()注册且第三个参数为true的事件处理程序中

捕获顺序： window->document->body->DOM->调用事件目标的父元素的捕获事件处理程序，在目标对象本事上注册的捕获事件处理程序不会被调用。
